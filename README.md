# 📌 REST API «Дошка оголошень» (Advanced: Security, Logging, Cloud Storage)

Це повноцінний RESTful API для керування приватними оголошеннями. У цій версії застосунок отримав архітектуру рівня Production: інтегровано багаторівневий захист, професійне структуроване логування та можливість завантаження медіафайлів у хмарне сховище.

---

## 🚀 Головні фічі та оновлення

- 🛡 **Базова безпека (Production-ready):**
  - **Helmet:** Автоматичне встановлення безпечних HTTP-заголовків (XSS захист, HSTS тощо).
  - **CORS:** Обмеження доступу до API лише для дозволених доменів (через `ALLOWED_ORIGINS`).
  - **Rate Limiting:** Захист від Brute-force атак. Обмеження до 10 запитів на 15 хвилин для маршрутів автентифікації (`/auth/*`).
- 📊 **Професійне логування:**
  - Використання **Pino** та **pino-http** для надшвидкого структурованого логування HTTP-запитів.
  - Трекінг бізнес-подій у контролерах (реєстрація, логін, створення оголошень, завантаження фото).
- 🖼 **Робота з медіафайлами:**
  - Підтримка `multipart/form-data` через **Multer** для тимчасового збереження файлів.
  - Інтеграція з **Cloudinary** для хмарного зберігання зображень. Локальні файли автоматично видаляються після вивантаження.
- 🔐 **Автентифікація та Авторизація:** JWT (Access + Refresh токени), Token Rotation, перевірка Ownership (редагування/видалення лише власних записів).
- 📚 **Swagger UI:** Автоматично згенерована документація (OpenAPI 3.0), яка підтримує завантаження файлів прямо з браузера.

## 🛠 Технологічний стек

| Технологія              | Роль                               |
| ----------------------- | ---------------------------------- |
| **Node.js & Express 5** | Серверне середовище та фреймворк   |
| **Prisma ORM & SQLite** | Робота з базою даних               |
| **JWT & Bcrypt**        | Безпека користувачів               |
| **Helmet & CORS**       | Захист HTTP-трафіку                |
| **Express Rate Limit**  | Захист від DDoS/Brute-force        |
| **Pino**                | Структуроване логування            |
| **Multer & Cloudinary** | Робота з файлами та хмарне сховище |

## 📦 Встановлення та запуск локально

1. **Склонуйте репозиторій:**
   ```bash
   git clone <посилання-на-репозиторій>
   cd <назва-папки>
   ```
   Встановіть залежності:

Bash
npm install
Налаштуйте змінні оточення (.env):

Фрагмент кода
DATABASE*URL="file:./dev.db"
JWT_SECRET="ваш*секретний*ключ"
JWT_REFRESH_SECRET="ваш*ключ_оновлення"

# CORS

ALLOWED_ORIGINS="http://localhost:5173,http://localhost:3000"

# Cloudinary (Для завантаження фото)

CLOUDINARY_CLOUD_NAME="your_cloud_name"
CLOUDINARY_API_KEY="your_api_key"
CLOUDINARY_API_SECRET="your_api_secret"
Міграція бази даних:

Bash
npm run prisma:migrate
npm run prisma:generate
Запуск сервера:

Bash
npm run dev
API працюватиме за адресою: http://localhost:3000

Документація: http://localhost:3000/api-docs
