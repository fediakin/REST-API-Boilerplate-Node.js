# 📌 REST API «Дошка оголошень» (з JWT-автентифікацією)

Сучасний та безпечний RESTful API для керування приватними оголошеннями. Проєкт реалізує повноцінну систему автентифікації на базі JSON Web Tokens (JWT) з механізмом ротації Refresh-токенів, строгу валідацію даних та перевірку прав власності (ownership) на ресурси.

---

## 🚀 Основні можливості

- 🔐 **Безпечна автентифікація:** Реєстрація та логін користувачів. Хешування паролів за допомогою `bcrypt`.
- 🔄 **JWT Token Rotation:** Використання короткострокових _Access_ токенів (15 хв) та довгострокових _Refresh_ токенів (7 днів), які надійно зберігаються в базі даних та передаються через `HttpOnly cookie`.
- 🛡 **Авторизація (Ownership):** Захист маршрутів `POST`, `PATCH`, `DELETE`. Користувач може редагувати або видаляти виключно власні оголошення.
- 📝 **Керування оголошеннями:** Створення, читання, оновлення та видалення (CRUD).
- 🔍 **Фільтрація та Пагінація:** Пошук за підрядком у назві, сортування (новіші/старіші), посторінковий вивід даних.
- ✅ **Валідація:** Сувора перевірка вхідних даних (Body, Params, Query) на рівні маршрутів за допомогою `celebrate` (на базі Joi).
- 📚 **Документація:** Автоматично згенерована інтерактивна документація Swagger UI (OpenAPI 3.0).

## 📦 Встановлення та запуск

1. **Склонуйте репозиторій:**
   ```bash
   git clone <посилання-на-ваш-репозиторій>
   cd <назва-папки>
   ```

Встановіть залежності:

Bash
npm install
Налаштуйте змінні оточення:
Створіть файл .env у корені проєкту та додайте секретні ключі:

Фрагмент кода
DATABASE*URL="file:./dev.db"
JWT_SECRET="ваш*супер*секретний*ключ*доступу"
JWT_REFRESH_SECRET="ваш*супер*секретний*ключ_оновлення"
Підготуйте базу даних (Міграція):

Bash
npm run prisma:migrate
npm run prisma:generate
Запустіть сервер (режим розробки):

Bash
npm run dev
Сервер працюватиме за адресою: http://localhost:3000

📖 Документація API (Swagger)
Інтерактивна документація генерується автоматично. Після запуску сервера перейдіть за адресою:
👉 http://localhost:3000/api-docs

Там ви зможете авторизуватися (кнопка Authorize) та протестувати всі ендпоінти прямо з браузера.

📡 Структура ендпоінтів
Автентифікація (/auth)
POST /auth/register — Реєстрація нового користувача.

POST /auth/login — Вхід та отримання токенів.

POST /auth/refresh — Оновлення пари токенів (Access + Refresh).

POST /auth/logout 🔒 — Вихід з системи (інвалідація токена).

GET /auth/me 🔒 — Отримання профілю поточного користувача.

Оголошення (/announcements)
GET /announcements — Отримання списку оголошень (з пагінацією та пошуком).

GET /announcements/:id — Отримання конкретного оголошення.

POST /announcements 🔒 — Створення нового оголошення.

PATCH /announcements/:id 🔒 — Часткове оновлення оголошення (лише для власника).

DELETE /announcements/:id 🔒 — Видалення оголошення (лише для власника).

(🔒 — вимагає валідний Bearer Token у заголовку Authorization)

🛡 Обробка помилок
Проєкт включає централізовану систему обробки помилок, яка повертає клієнту чіткий JSON:

400 Bad Request — Помилки валідації Celebrate або некоректний JSON.

401 Unauthorized — Невірні облікові дані, відсутній або прострочений JWT токен.

403 Forbidden — Спроба змінити/видалити чуже оголошення.

404 Not Found — Ресурс або маршрут не знайдено (зокрема Prisma P2025).

409 Conflict — Користувач з таким username вже існує.

500 Internal Server Error — Непередбачені помилки сервера.
